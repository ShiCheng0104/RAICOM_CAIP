# 方向一：欺诈交易智能识别答辩方案 v7

## 1. v7 新增闭环

v7 在 v6 的图分析团伙挖掘、Kafka + Flink 实时检测、多模型对比和 Dashboard 基础上，继续补齐赛题评分中容易丢分的三类能力：

```text
1. 可调参仿真环境
2. 自适应学习与模型生命周期管理
3. 安全审计、混合云和高并发低延迟验证材料
```

## 2. 可调参仿真环境

新增命令：

```powershell
python -m fraudsim.simulator.generate --dataset fp_fraudsim_injected --rows 10000 --fraud-ratio 0.04 --fraud-group-count 20 --fraud-group-size 12
```

支持配置：

```text
欺诈比例、团伙数量、团伙规模、共享设备强度、共享 IP 强度、跨境比例、大额比例、夜间比例、交易速率
```

输出：

```text
transaction_stream.jsonl
simulation_labels.parquet
simulation_config.json
```

答辩价值：

```text
项目不再只是离线数据回放，而是具备可调参的仿真交易流生成能力，能够模拟跨时段、跨渠道和团伙协作欺诈场景。
```

## 3. 自适应学习与模型迭代

新增闭环：

```text
Dashboard 人工审核
  -> POST /feedback
  -> Kafka feedback_events
  -> data/processed/{dataset}/feedback/feedback_pool.jsonl
  -> train_with_feedback
  -> models/{model}/candidate
  -> /models/promote
  -> models/{model}/latest
  -> /models/rollback
```

新增接口：

| 接口 | 作用 |
|---|---|
| `GET /models/{model}/candidate` | 查看候选模型指标 |
| `POST /models/promote` | 发布 candidate 为 latest |
| `POST /models/rollback` | 回滚到上一版 latest |
| `GET /audit/recent` | 查看模型切换、反馈、重训等审计日志 |

答辩价值：

```text
系统具备“反馈采集、候选训练、指标评估、受控发布、快速回滚”的模型迭代机制，避免直接用人工反馈覆盖线上模型。
```

## 7. “仍可增强”实施结果

### 7.1 GraphSAGE 图 embedding 旁路实验

新增 `fraudsim.graphsage_experiment`，对统一交易图做受控采样，使用两层 GraphSAGE 生成节点 embedding 和节点风险分。

最终本机实验：

| 指标 | 数值 |
|---|---:|
| 采样节点 | 30,000 |
| 采样边 | 25,732 |
| 节点欺诈比例 | 21.32% |
| PR-AUC | 0.6799 |
| ROC-AUC | 0.7855 |
| F1@0.5 | 0.5881 |

首轮实验发现欺诈节点优先采样导致欺诈率接近 100%、指标虚高，随后修正为混合随机节点采样和受控正样本比例。最终结果更可信。GraphSAGE 当前保持旁路实验，不直接改变线上 LightGBM 决策。

### 7.2 Dashboard 阈值沙盒

训练流程新增 `evaluation_scores.parquet`，已有模型可通过 `fraudsim.training.build_scorecard` 补建。

指标页现在支持拖动：

```text
人工审核阈值
高风险拦截阈值
```

并实时展示：

```text
放行量、审核量、拦截量、拦截精度、欺诈召回、F1、FPR、误拦量、漏过量
```

阈值调优从固定文档指标升级为可交互业务策略沙盒。

使用人工审核阈值 `0.50`、高风险拦截阈值 `0.80` 的实测结果：

| 指标 | 数值 |
|---|---:|
| 测试交易 | 123,649 |
| 放行 | 112,249 |
| 人工审核 | 1,647 |
| 高风险拦截 | 9,753 |
| 拦截精度 | 96.08% |
| 欺诈召回 | 87.69% |
| FPR | 0.338% |

### 7.3 集中审计日志

敏感操作继续写入本地 `models/audit/audit.jsonl`，同时异步写入 Kafka `audit_events`。Kafka 不可用时，本地日志仍作为保底审计记录。

### 7.4 Kubernetes 混合云演示

新增：

```text
deploy/k8s/edge.yaml
deploy/k8s/center.yaml
deploy/k8s/README.md
```

边缘侧包含 Kafka、Redis、双副本 Model API、Flink 风险作业、PVC、Secret 和 NetworkPolicy；中心侧包含反馈候选训练和 GraphSAGE 旁路实验 CronJob。

当前本机未启用 Kubernetes 集群，因此未宣称完成真实多节点集群部署。清单已通过单元结构测试和 kubeconform 严格 schema 校验：共 20 个资源，20 个有效、0 个无效、0 个错误。真实部署时还需配置镜像仓库、PVC/对象存储、MirrorMaker 2 和 TLS/SASL。
