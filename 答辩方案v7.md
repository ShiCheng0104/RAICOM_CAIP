# 方向一：欺诈交易智能识别答辩方案 v7

## 1. v7 新增闭环

v7 在 v6 的图分析团伙挖掘、Kafka + Flink 实时检测、多模型对比和 Dashboard 基础上，继续补齐赛题评分中容易丢分的三类能力：

```text
1. 可调参仿真环境
2. 自适应学习与模型生命周期管理
3. 安全审计、混合云和高并发低延迟验证材料
```

## 2. 可调参仿真环境

新增命令：

```powershell
python -m fraudsim.simulator.generate --dataset fp_fraudsim_injected --rows 10000 --fraud-ratio 0.04 --fraud-group-count 20 --fraud-group-size 12
```

支持配置：

```text
欺诈比例、团伙数量、团伙规模、共享设备强度、共享 IP 强度、跨境比例、大额比例、夜间比例、交易速率
```

输出：

```text
transaction_stream.jsonl
simulation_labels.parquet
simulation_config.json
```

答辩价值：

```text
项目不再只是离线数据回放，而是具备可调参的仿真交易流生成能力，能够模拟跨时段、跨渠道和团伙协作欺诈场景。
```

## 3. 自适应学习与模型迭代

新增闭环：

```text
Dashboard 人工审核
  -> POST /feedback
  -> Kafka feedback_events
  -> data/processed/{dataset}/feedback/feedback_pool.jsonl
  -> train_with_feedback
  -> models/{model}/candidate
  -> /models/promote
  -> models/{model}/latest
  -> /models/rollback
```

新增接口：

| 接口 | 作用 |
|---|---|
| `GET /models/{model}/candidate` | 查看候选模型指标 |
| `POST /models/promote` | 发布 candidate 为 latest |
| `POST /models/rollback` | 回滚到上一版 latest |
| `GET /audit/recent` | 查看模型切换、反馈、重训等审计日志 |

答辩价值：

```text
系统具备“反馈采集、候选训练、指标评估、受控发布、快速回滚”的模型迭代机制，避免直接用人工反馈覆盖线上模型。
```

## 4. 安全与合规

新增安全能力：

```text
1. 设置 FRAUDSIM_API_KEY 后，/predict、/feedback、/reload、/demo/run、模型发布/回滚等敏感接口需要 X-API-Key
2. 敏感操作写入 models/audit/audit.jsonl
3. Dashboard 对长 ID 做标签化展示和滚动显示，生产环境可进一步哈希化
4. 文档说明仿真数据、访问控制、审计日志、模型发布备份和混合云传输安全
```

安全文档：

```text
docs/security_design.md
```

## 5. 混合云和性能验证

新增文档：

```text
docs/hybrid_cloud_deployment.md
```

最小验证口径：

```text
Docker Compose 模拟边缘实时侧：Kafka、Flink、Redis、Model API
本地 Python 模拟中心训练侧：离线训练、图挖掘、压测客户端
```

验证指标：

```text
API P50/P95/P99 延迟
Kafka 输入速率
Flink 输出速率
端到端耗时
错误率
高风险告警数
```

本机 API 延迟实测：

| 指标 | 数值 |
|---|---:|
| 单笔平均延迟 | 115.78 ms |
| 单笔 P95 | 156.55 ms |
| 单笔 P99 | 240.30 ms |
| 微批单条折算平均延迟 | 5.00 ms |
| 微批单条折算 P95 | 6.16 ms |
| 平均单条吞吐收益 | 23.16x |

## 6. 与评分项对应

| 评分点 | 当前实现 |
|---|---|
| 仿真环境与实时检测能力 | 可调参仿真器 + JSONL/Kafka 回放 + 实时风险评分 |
| 流式数据处理框架 | Kafka + PyFlink + Redis + FastAPI |
| 自适应学习与模型迭代 | feedback_pool + candidate/latest/rollback |
| 多维度特征构建 | 交易、窗口、画像、设备/IP、商户、图统计、团伙挖掘 |
| 创新性 | 可解释图分析团伙挖掘 + 团伙证据图 + 反馈驱动迭代 |
| 实用性 | Docker 化、一键启动、Dashboard 工作台、模型热插拔 |
| 成熟度 | 单元测试、压测脚本、启动脚本、README 和专题文档 |
| 安全性 | API Key、审计日志、数据脱敏建议、混合云安全设计 |
