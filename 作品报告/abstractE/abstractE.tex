\section*{\centering\bfseries\zihao{4}ABSTRACT}
\addcontentsline{toc}{section}{Abstract}

\vspace{0.5cm}

This report presents FP-FraudSim, a streaming fraud detection system for Task 1. It integrates configurable transaction simulation, Kafka/Flink real-time scoring, risk decisioning, fraud-ring tracing, and a human-feedback model iteration loop.

The system combines transaction, profile, window, and graph features, and outputs risk levels, actions, and reason codes. It further provides a threshold sandbox, a GraphSAGE sidecar experiment, controlled candidate-model promotion and rollback, API-key protection, centralized auditing, and Docker Compose/Kubernetes deployment artifacts. In the recorded benchmark, micro-batch inference reduced amortized per-record latency to 5.00 ms and achieved a 23.16x throughput gain over single-record requests.

\noindent\textbf{Key Words: }financial anti-fraud; transaction fraud detection; Kafka; Flink; graph mining; model iteration
