# 鸿蒙星盾相关论文整理

## 说明

目前以“星盾安全架构”直接命名的公开学术论文较少，公开资料更多集中在 HarmonyOS/OpenHarmony 的微内核、安全架构、应用安全、静态分析和隐私保护等方向。因此，这份整理分为两部分：

- 与星盾底层技术强相关的论文
- 官方技术资料与白皮书

如果要研究“鸿蒙星盾”本身，建议把官方星盾文章作为总览，再结合下列论文理解其底层技术支撑。

## 一、与鸿蒙星盾强相关的论文

### 1. Microkernel Goes General: Performance and Compatibility in the Hong-Meng Production Microkernel

- 来源：OSDI 2024 / USENIX
- 论文链接：https://www.usenix.org/system/files/osdi24-chen-haibo.pdf
- 相关性：极高
- 核心概要：这篇论文围绕鸿蒙生产级微内核展开，重点讨论微内核在通用场景下如何同时满足性能、兼容性和系统隔离要求。论文核心价值在于解释鸿蒙为什么能在保留微内核安全优势的同时，适配更复杂的终端场景。
- 与星盾的关系：星盾强调“微内核隔离、最小特权、纵深防御”，这篇论文可以看作星盾底层可信内核能力的重要理论和工程支撑。

### 2. HM-SAF: Cross-Layer Static Analysis Framework For HarmonyOS

- 来源：2023 IEEE Smart World Congress (SWC)
- DOI：https://doi.org/10.1109/SWC57546.2023.10449022
- 论文链接：https://ieeexplore.ieee.org/document/10449022
- 相关性：高
- 核心概要：论文提出面向 HarmonyOS 应用的跨层静态分析框架 HM-SAF，重点解决现有框架难以完成跨层安全分析的问题。其目标是支持对 HarmonyOS 应用进行数据泄漏等安全风险检测。
- 与星盾的关系：星盾在应用层强调权限约束、行为审计和主动防御，这篇论文对应的是“如何分析鸿蒙应用是否存在安全问题”的工具链能力。

### 3. 鸿蒙应用安全与个人信息保护思考

- 来源：信息通信技术与政策，2024 年第 50 卷第 6 期
- DOI：https://doi.org/10.12267/j.issn.2096-5931.2024.06.010
- 论文链接：http://ictp.caict.ac.cn/CN/10.12267/j.issn.2096-5931.2024.06.010
- 相关性：高
- 核心概要：文章从应用开发、发布、运行三个阶段出发，分析 HarmonyOS 应用在安全和个人信息保护方面面临的风险，并提出全生命周期安全保护思路。
- 与星盾的关系：这篇论文对理解星盾在应用生态中的作用很关键，尤其是权限治理、隐私保护、应用分发安全等方面。

### 4. ArkAnalyzer: The Static Analysis Framework for OpenHarmony

- 来源：arXiv，2025
- DOI：https://doi.org/10.48550/arXiv.2501.05798
- 论文链接：https://arxiv.org/abs/2501.05798
- PDF：https://arxiv.org/pdf/2501.05798
- 相关性：高
- 核心概要：该论文提出 OpenHarmony 的静态分析框架 ArkAnalyzer，解决 ArkTS 程序分析基础设施不足的问题，并将其开源，便于开发者实现漏洞检测、代码质量分析和自动修复等工作。
- 与星盾的关系：虽然它是 OpenHarmony 方向论文，但它补足了鸿蒙安全工具链层面的研究基础，适合从“安全分析框架”角度理解星盾生态的技术支撑。

### 5. HarmonyFlow: 基于方舟Panda IR的HarmonyOS应用静态分析框架

- 来源：软件学报，在线优先发表
- DOI：http://dx.doi.org/10.13328/j.cnki.jos.007477
- 论文链接：https://jos.org.cn/josen/article/abstract/7477
- PDF：https://jos.org.cn/josen/article/pdf/7477
- 相关性：高
- 核心概要：论文针对 HarmonyOS 应用静态分析仍不成熟的问题，提出基于 Panda IR 的静态分析框架 HarmonyFlow。论文强调，在真实安全检测场景中很难拿到应用源码，因此需要基于中间表示做更可落地的分析。
- 与星盾的关系：这篇论文非常适合和 HM-SAF 一起看，前者更偏跨层安全分析，后者更偏基于 Ark/Panda IR 的工程化静态分析能力。

### 6. Software Engineering for OpenHarmony: A Research Roadmap

- 来源：ACM Digital Library
- 论文链接：https://dl.acm.org/doi/10.1145/3720538
- 相关性：中高
- 核心概要：这篇论文从软件工程视角总结 OpenHarmony 的研究问题和未来路线，涵盖生态、兼容性、质量、测试、分析等多个方向。
- 与星盾的关系：它不是专门的安全论文，但适合作为研究入口。通过它可以快速了解 OpenHarmony 整体研究版图，再定位安全相关方向。

### 7. HmTest: Automated Testing of HarmonyOS Apps via Model-Driven Navigation and Reinforcement Learning

- 来源：Journal of Computer Science and Technology，2025
- 论文链接：https://jcst.ict.ac.cn/article/doi/10.1007/s11390-025-5142-4
- 相关性：中
- 核心概要：论文研究 HarmonyOS 应用自动化测试，结合模型驱动导航和强化学习提升测试覆盖率，并用静态分析构建页面迁移图来指导测试。
- 与星盾的关系：这篇论文更偏测试与质量保障，但如果后续研究“如何验证鸿蒙安全机制是否稳定有效”，这篇可以作为补充阅读。

## 三、推荐阅读顺序

1. 先读“星盾安全架构”官方文章，建立六层防御框架认知。
2. 再读 OSDI 2024 的微内核论文，理解底层隔离和可信基础。
3. 接着读“鸿蒙应用安全与个人信息保护思考”，补齐应用生态和隐私保护视角。
4. 然后读 HM-SAF、ArkAnalyzer、HarmonyFlow，进入安全分析工具链研究。
5. 最后读 OpenHarmony 路线图和 HmTest，补充生态研究与验证方法。

## 四、一句话总结

如果把“星盾”看作鸿蒙的安全总框架，那么它背后的核心研究支撑可以概括为三条线：微内核与可信基础、应用安全与隐私保护、静态分析与自动化安全工具链。
