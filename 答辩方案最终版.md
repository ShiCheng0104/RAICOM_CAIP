# 方向一：欺诈交易智能识别答辩方案 最终版

## 1. 项目定位

本项目面向第三方支付全链路风险防控，构建了一套从多源异构数据融合、交易仿真、离线训练，到 Kafka + Flink 实时检测、团伙链路挖掘、人工审核和模型迭代的完整系统。

系统在交易发生时输出：

```text
风险分数 risk_score
风险等级 risk_level
处置决策 decision
可解释理由码 reason_codes
图分析团伙证据
```

项目主张不是单独训练一个分类模型，而是形成可持续运行的风控闭环：

```text
数据构建 -> 模型训练 -> 实时检测 -> 人工反馈 -> 候选模型 -> 受控发布与回滚
```

## 2. 系统总体架构

```text
公开数据集与仿真交易
  -> 统一字段、统一实体、统一时间轴、统一标签
  -> 交易流 + 用户/设备/IP/商户画像 + 图节点/图边

离线训练侧
  -> LightGBM 主模型
  -> XGBoost / CatBoost / sklearn 模型对比
  -> GraphSAGE 图 embedding 旁路实验
  -> candidate / latest / rollback

实时检测侧
  -> Kafka transaction_events
  -> PyFlink event time + watermark + 窗口特征
  -> Redis 静态画像与图风险画像
  -> FastAPI 微批推理
  -> risk_results / alert_events / late_events

研判与迭代侧
  -> Dashboard 实时结果、试算、阈值沙盒、图证据
  -> 人工审核 feedback_events
  -> 候选模型重训、评估、发布与回滚
  -> 本地审计 + Kafka audit_events
```

## 4. 数据构建与异构融合

系统将多个公开金融欺诈数据集映射为统一交易语义，解决不同字段、实体命名、时间窗口和标签口径不一致的问题。

核心方法：

```text
字段异构：映射到统一交易、主体、渠道、设备、IP 和商户语义
实体冲突：使用来源前缀构造全局唯一 ID
时间异构：转换为统一时间轴并按时间切分 train / valid / test
标签异构：统一为 is_fraud = 0 / 1，未知标签不进入监督训练
窗口差异：离线按事件时间构建历史窗口，在线由 Flink 使用同口径计算
数据泄露：排除标签衍生风险率、规则分和直接标签字段
```

统一数据最终形成：

```text
交易事实表
用户画像
设备画像
IP 地理与风险画像
商户画像
图节点与图边
训练、验证、测试时间切分
可回放 transaction_stream.jsonl
```

## 5. 多维特征与模型

特征体系覆盖：

| 特征类型 | 作用 |
|---|---|
| 交易基础特征 | 表达金额、渠道、支付方式、跨境和时间属性 |
| 时间窗口特征 | 识别短时高频、突发金额和收款方扩散 |
| 用户画像 | 表达长期金额、活跃度和常用环境 |
| 设备/IP 特征 | 识别共享设备、代理 IP 和多账号操作 |
| 商户画像 | 识别集中交易、投诉和异常商户行为 |
| 图统计与团伙特征 | 识别共享资源、欺诈邻居和协作式欺诈 |

LightGBM 作为线上主模型，因为其训练速度快、适合表格特征、推理延迟低，并且便于解释和热更新。其他模型通过统一 adapter 接口完成训练和比较。

主模型实测：

|---|---:|
| 测试集交易数 | 123,649 |
| 特征数 | 67 |
| PR-AUC | 0.9727 |
| ROC-AUC | 0.9949 |
| F1 | 0.9169 |
| 高风险阈值 FPR | 0.338% |

## 6. 实时检测与性能


## 7. 图分析与 GraphSAGE

图分析模块从共享设备、共享 IP、商户、收款方和历史欺诈种子中发现疑似团伙，并输出：

```text
团伙风险分
成员用户与共享资源
证据类型与理由码
交互式证据子图
实体到团伙反查链路
不同风险阈值下的精度与召回
```

图挖掘效果：

```text
检测候选团伙：475
高置信阈值：0.65
高置信交易精度：98.71%
注入团伙总体召回：77.93%
```

GraphSAGE 旁路实验使用受控图采样和两层邻居聚合：

| 指标 | 数值 |
|---|---:|
| 采样节点 | 30,000 |
| 采样边 | 25,732 |
| PR-AUC | 0.6799 |
| ROC-AUC | 0.7855 |
| F1@0.5 | 0.5881 |

GraphSAGE 当前不直接进入线上决策。它作为创新性和后续实验路径存在，避免未经充分时序验证的 embedding 影响稳定的 LightGBM 主模型。

## 8. 阈值策略与业务研判

Dashboard 指标页提供阈值沙盒，可调整：

```text
人工审核阈值
高风险拦截阈值
```

并即时计算放行量、审核量、拦截量、精度、召回、F1、FPR、误拦量和漏过量。

阈值 `medium=0.50`、`high=0.80` 实测：

| 指标 | 数值 |
|---|---:|
| 放行 | 112,249 |
| 人工审核 | 1,647 |
| 高风险拦截 | 9,753 |
| 拦截精度 | 96.08% |
| 欺诈召回 | 87.69% |
| FPR | 0.338% |

阈值不是赛题固定值，而是结合业务误报成本、欺诈损失和人工审核容量进行调优。

## 9. 自适应学习与模型生命周期

当前迭代闭环：

```text
Dashboard 人工审核
  -> feedback_events + feedback_pool.jsonl
  -> 使用新增标签重训 candidate
  -> 查看候选模型指标
  -> 人工确认后 promote
  -> latest 在线加载
  -> 异常时 rollback
```

系统不会在交易流输入时直接覆盖线上模型，而是采用候选模型、指标评估、受控发布和快速回滚，以降低在线学习导致模型退化的风险。

项目已经完成赛题方向一要求的核心闭环：能够构造多源异构仿真交易环境，通过 Kafka + Flink 实时计算多维特征并完成风险判断，同时具备多模型训练、可解释团伙挖掘、人工反馈迭代、阈值策略调优、安全审计和混合云部署设计。
